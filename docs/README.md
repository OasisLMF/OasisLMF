# OasisLMF documentation

This is the in-repo Sphinx documentation for the `oasislmf` package. It is
**co-located with the code** so that a change in behaviour and its documentation
travel together in the same pull request. See
[`DOCS_STRATEGY.md`](../DOCS_STRATEGY.md) for the overall documentation strategy
and the consolidation plan this structure is part of.

## Structure (Diátaxis)

The docs are organised using the [Diátaxis](https://diataxis.fr/) framework —
each page has one job:

```
source/
  index.rst                 landing page (audience cards + section toctrees)
  tutorials/                learning-oriented, "learn by doing"
  how-to/ (installation, building-and-running-models, logging-configuration)
                            task-oriented recipes
  reference/                information-oriented, mostly generated from source
    index.md                config, env vars, and the Python API
    api/                    auto-generated by sphinx-autoapi (do not edit)
  explanation/              understanding-oriented, co-located with the code
    financial-module.rst    the FM conceptual model (migrated from GenerateDocs)
    fm-architecture.md      the FM Python implementation (was orphaned in-repo)
  _static/images/           figures
```

New pages should prefer **Markdown (MyST)**; existing `.rst` is migrated
incrementally — both compile.

## Requirements

The documentation dependencies are declared in the repository-root
`requirements.in` (`furo`, `sphinx-autoapi`, `myst-parser`, `sphinx-design`,
`sphinx-copybutton`). Install the package and the docs deps into your
environment:

```sh
pip install -e .
pip install furo sphinx-autoapi myst-parser sphinx-design sphinx-copybutton
```

Also make sure `ods_tools` is available (the development branch has the latest
features):

```sh
pip install git+https://git@github.com/OasisLMF/ODS_Tools
```

## Build

From the repository root:

```sh
./docs/build.sh
```

The HTML is written to `docs/build/html/`; open `docs/build/html/index.html` in a
browser.

Or directly with Sphinx (run from the `docs/` directory so the generated CLI
options file lands in the right place):

```sh
cd docs
sphinx-build -b html source build/html
```

### Live preview

For an auto-reloading local preview while editing:

```sh
pip install sphinx-autobuild
cd docs
sphinx-autobuild source build/html
```

### Check for dead links

```sh
cd docs
make linkcheck        # or: sphinx-build -b linkcheck source build/linkcheck
```

## The API reference (sphinx-autoapi)

The Python API reference is generated from source docstrings by `sphinx-autoapi`,
which is **AST-based** — it parses the source and never imports the package, so
the heavy runtime dependencies (numba, etc.) are not imported at build time.

`autoapi_dirs` in `source/conf.py` controls the scope. It is currently **scoped
to the Financial Module** (`oasislmf/pytools/fm`) so the reference sits beside
its explanation pages and the build takes ~20 seconds instead of the ~4 minutes
(and ~500 unstructured pages) of the previous whole-package configuration. Widen
it per subsystem as each adopts the reference/explanation pattern.

## Automation

- `build-docs.yml` builds the docs for every PR targeting `main` and uploads the
  HTML as a downloadable artifact.
- `build-and-deploy-docs.yml` builds and deploys to GitHub Pages on pushes to
  `main`.
